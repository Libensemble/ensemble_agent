# libEnsemble Generator Functions

libEnsemble supports two styles of generator configuration:

- **Traditional generators** — libEnsemble-native gen functions configured via `gen_f`, explicit `inputs`/`outputs`, and `user` dicts with bounds/parameters.
- **VOCS generators (gest-api)** — External optimization libraries (Xopt, Optimas) configured via a VOCS object that defines variables, objectives, and constraints. The generator is passed as an object rather than a function reference.

## When to Choose a Generator Style

Use **VOCS generators** when the user:
- Mentions Xopt, Optimas, or a specific Xopt generator (e.g., Expected Improvement, Nelder-Mead, MOBO)
- Wants to use a VOCS-based configuration for variables and objectives
- Needs generators from external optimization frameworks

Use **traditional generators** for all other cases (APOSMM, sampling, gpCAM, etc.).

## When to Choose a Generator

| Goal | Generator | Style | Module / Package |
|------|-----------|-------|------------------|
| Find multiple minima in a space | APOSMM | Traditional | `persistent_aposmm` |
| Simple random/uniform exploration | Persistent sampling | Traditional | `persistent_sampling` |
| Sampling with variable compute resources | Var-resource sampling | Traditional | `persistent_sampling_var_resources` |
| Bayesian optimization with surrogate | gpCAM | Traditional | `persistent_gpCAM` |
| Bayesian optimization (Ax) | Ax Multitask | Traditional | (community) |
| Multi-objective optimization | DEAP-NSGA-II, VTMOP | Traditional | (community) |
| Uniform then local optimization | uniform_or_localopt | Traditional | `uniform_or_localopt` |
| Bayesian optimization (Xopt) | Expected Improvement, Upper Confidence Bound, etc. | VOCS | `xopt` |
| Simplex optimization (Xopt) | Nelder-Mead | VOCS | `xopt` |
| Multi-objective Bayesian (Xopt) | MOBO | VOCS | `xopt` |
| Optimization (Optimas) | Various | VOCS | `optimas` |

If the user says "find minima", "optimization", "multiple local minima" -> APOSMM.
If the user says "sample", "explore", "sweep", "sensitivity" -> persistent sampling.
If the user says "surrogate", "Gaussian process", "Bayesian" -> gpCAM or Xopt Expected Improvement.
If the user says "Xopt", "VOCS", "Optimas", or names a specific Xopt generator -> VOCS style.

## Sampling Generators

### persistent_sampling (persistent_uniform)
Random uniform sampling across parameter space. After the initial batch, creates p new random points for every p points returned.

gen_specs["user"]: `lb`, `ub`, `initial_batch_size`
gen_specs outputs: `x (float, n)`

### persistent_sampling_var_resources (uniform_sample_with_procs_gpus)
Uniform sampling that also assigns CPU/GPU resources per simulation.

gen_specs["user"]: `initial_batch_size`, `max_procs`, `lb`, `ub`
gen_specs outputs: `num_procs (int)`, `num_gpus (int)`, `x (float, 2)`

## Optimization Generators

### APOSMM (persistent_aposmm)
See the dedicated APOSMM skill file for full details.
Asynchronously Parallel Optimization Solver for finding Multiple Minima.
Allocator: `persistent_aposmm_alloc`
Requires one persistent generator worker (nworkers = sim_workers + 1).

### uniform_or_localopt
Samples uniformly in non-persistent mode, then runs NLopt local optimization in persistent mode. Good for simple optimize-after-sampling workflows.

## Modeling / Approximation Generators

### gpCAM (persistent_gpCAM)
Gaussian Process-based adaptive sampling. Best for expensive simulations where intelligent exploration with a surrogate model is valuable.

gen_specs["user"]: `batch_size`, `lb`, `ub`
Variant: `persistent_gpCAM_covar` adds `use_grid: True` and `final_gen_send: True`.

### Surmise
Modular Bayesian calibration/inference framework. Demonstrates cancelling previously issued simulations.

### Tasmanian
Evaluates points generated by the Tasmanian sparse grid library.

## VOCS Generators (gest-api)

VOCS generators use external optimization libraries (Xopt, Optimas) via the gest-api interface. Configuration uses a VOCS object to define the optimization problem and a generator object from the external library.

### Key differences from traditional generators

- Variables are named individually in VOCS (`{"x0": [lb, ub], "x1": [lb, ub]}`) instead of a single `"x"` array.
- Objectives and constraints are named in VOCS (`{"f": "MINIMIZE"}`).
- H fields match VOCS names — sim functions receive `H["x0"]`, `H["x1"]` (not `H["x"]`).
- GenSpecs uses `generator=`, `vocs=`, and `batch_size=` instead of `gen_f`, `inputs`/`outputs`/`user`.
- SimSpecs uses `vocs=` instead of explicit `inputs`/`outputs`.
- Allocator: `start_only_persistent.only_persistent_gens` (not `persistent_aposmm_alloc`).
- No `add_random_streams()` needed.

### Initial sampling

Xopt generators require evaluated data before they can suggest points. Set `initial_sample_method` in GenSpecs to have libEnsemble produce and evaluate an initial sample before starting the generator:

- `initial_sample_method="uniform"` — uniform random sample from VOCS bounds
- `initial_batch_size` — required, specifies how many sample points to produce

Generators that handle their own sampling (libEnsemble native generators, Optimas) do not need this.

### Sim function adaptation

When using VOCS generators with an executor-based sim function, the sim must read individual variable names from H rather than unpacking `H["x"]`. The `input_names` in `sim_specs["user"]` should match the VOCS variable names directly. Use `get_examples(collection="vocs")` and `get_example(name, collection="vocs")` to see reference implementations.

### Xopt generators

Xopt provides generators for Bayesian optimization, simplex methods, and more. Import from `xopt.generators`.

Common generators:
- `ExpectedImprovementGenerator` — Bayesian optimization, supports parallel batches
- `UpperConfidenceBoundGenerator` — Bayesian optimization with exploration/exploitation tuning
- `NelderMeadGenerator` — Simplex-based local optimization (batch_size=1)
- `MOBOGenerator` — Multi-objective Bayesian optimization

```python
from gest_api.vocs import VOCS
from xopt.generators.bayesian.expected_improvement import ExpectedImprovementGenerator

vocs = VOCS(
    variables={"x0": [lb, ub], "x1": [lb, ub]},
    objectives={"f": "MINIMIZE"},
)

gen = ExpectedImprovementGenerator(vocs=vocs)

gen_specs = GenSpecs(
    generator=gen,
    initial_batch_size=num_workers,
    initial_sample_method="uniform",
    batch_size=num_workers,
    vocs=vocs,
)
sim_specs = SimSpecs(sim_f=my_sim, vocs=vocs)
alloc_specs = AllocSpecs(alloc_f=only_persistent_gens)
```

Use `get_examples(collection="vocs")` to see available VOCS examples, and `get_example(name, collection="vocs")` to fetch the full source.
